# 📊 Анализ e-commerce платформы (Olist)

## 📌 О проекте

В данном проекте проведён анализ данных бразильской e-commerce платформы Olist.

Цель проекта — изучить ключевые метрики бизнеса, поведение клиентов и качество сервиса (включая доставку и отзывы).

Проект выполнен с использованием SQL и имитирует задачи, с которыми сталкивается аналитик данных в реальной работе.

## 🧹 Подготовка данных

В исходных данных были обнаружены проблемы качества:

* даты хранились в текстовом формате
* отсутствующие значения были представлены как пустые строки (`''`), а не `NULL`

Это могло привести к некорректным расчётам и ошибкам при приведении типов.

В рамках подготовки данных:

* пустые строки были заменены на `NULL`
* столбцы с датами приведены к типу `timestamp`
* проведена базовая проверка структуры данных и связей между таблицами

После очистки данные стали пригодны для дальнейшего анализа.

## 🧱 Построение витрины данных

Для корректного анализа была построена витрина на уровне заказа.

Проблема исходных данных заключалась в том, что:

* один заказ может содержать несколько товаров
* один заказ может иметь несколько платежей

Это приводило к дублированию строк при объединении таблиц.

Решение:

* предварительная агрегация данных по заказам
* объединение агрегированных таблиц

В результате получена витрина, где каждая строка соответствует одному заказу, что позволяет корректно рассчитывать бизнес-метрики.

## 📈 Анализ метрик

На основе подготовленной витрины были рассчитаны ключевые бизнес-метрики.

### Основные метрики

* общая выручка
* средний чек (AOV)
* среднее количество товаров в заказе

Для корректности расчётов использовались только завершённые заказы (`order_status = 'delivered'`).

### Динамика

Проведен анализ метрик по месяцам:

* количество заказов
* выручка
* средний чек
* среднее количество товаров

Это позволило оценить изменения бизнес-показателей во времени.
